현장 위험성 평가 결과 디지털화 및 개선

시스템 구축 배경

기존의 위험성 평가는 종이 문서로 작성되어 데이터의 축적과 활용이 어려웠습니다. 현장 관리자가 수기로 작성한 문서는 분실 위험이 있고, 통계 처리를 위해서는 다시 엑셀로 입력해야 하는 이중 작업이 발생했습니다.

시스템 구축 전후 비교

Before (기존)

종이 문서 작성 -> 수거 -> 엑셀 재입력 (비효율적)

After (개선)

모바일/태블릿 직접 입력 -> DB 자동 저장 (효율적)

기술 스택 및 구현

  • Frontend: HTML5, CSS3, Vanilla JavaScript (별도의 프레임워크 없이 경량화 구현)
  • Backend: Python Flask (간단한 API 서버)
  • Database: SQLite (로컬 테스트용) / MySQL (배포용)

반응형 웹 디자인을 적용하여 현장에서 태블릿이나 스마트폰으로 즉시 접속하여 위험 요인을 입력하고 사진을 업로드할 수 있도록 구현했습니다.
